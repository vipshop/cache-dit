# Cache-DiT Serving

Cache-DiT is already fully integrates with [SGLang Diffusion](https://docs.sglang.io/diffusion/performance/cache/cache_dit.html) and [vLLM-Omni](https://docs.vllm.ai/projects/vllm-omni/en/latest/user_guide/diffusion/cache_acceleration/cache_dit/). Thus, we highly <span style="color:#c77dff;">recommend</span> using Cache-DiT through the serving capabilities of [SGLang Diffusion](https://docs.sglang.io/diffusion/performance/cache/cache_dit.html) and [vLLM-Omni](https://docs.vllm.ai/projects/vllm-omni/en/latest/user_guide/diffusion/cache_acceleration/cache_dit/). Cache-DiT's own serving module will no longer be maintained and already deprecated after Cache-DiT v1.3.5. <i>Let's go beyond Cache-DiT!</i>

- [SGLang Diffusion](https://docs.sglang.io/diffusion/performance/cache/cache_dit.html)
- [vLLM-Omni](https://docs.vllm.ai/projects/vllm-omni/en/latest/user_guide/diffusion/cache_acceleration/cache_dit/)

## Simple HTTP Server

Deprecated: The simple HTTP server provided by Cache-DiT is deprecated and will be removed in future releases. We recommend using the serving capabilities of [SGLang Diffusion](https://docs.sglang.io/diffusion/performance/cache/cache_dit.html) and [vLLM-Omni](https://docs.vllm.ai/projects/vllm-omni/en/latest/user_guide/diffusion/cache_acceleration/cache_dit/) for production use.
